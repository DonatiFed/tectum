Reonic AI Solar Designer

Pipeline Summary & Evaluation — April 2026

What it does

A 10-step engine that takes customer inputs and generates 3 solar + battery + heat pump options, each with a full Bill of Materials and a personalized AI explanation.

Customer Input → [1] Context → [2] Candidates → [3] Physics+Economics → [4] Realism Score → [5] Select 3 Options → [6] Match Products → [7] Build BOM → [8] Validate → [9] Format → [10 AI] LLM Explanation

Components sized: PV modules, battery, wallbox (if EV), Vaillant heat pump (if gas/oil heating, on Max Independence).
Speed: 3.6ms per run. Stateless — every input change re-runs the full pipeline.
AI layer: Local Llama 3.2 (3B) or Claude generates personalized customer explanation. Not an email — inline content for the offer page.

Backtest Robustness: Three Roof-Cap Strategies

Historical roof geometry was unavailable, so Backtest A uses observed module count + 4 as a synthetic roof-capacity proxy. To verify this isn’t inflating accuracy, we also test with a fixed 30-module cap (B) and demand-based caps (C). In production, the 3D roof parser provides the real constraint.

Key finding: The roof constraint is the single most important input — with the real roof (Backtest A), MAE is 1.5 kWp. Without it, the optimizer oversizes because it has more headroom. This confirms that in production (where the 3D roof parser provides real constraints), accuracy will be high. Battery and brand accuracy are stable across all strategies, proving those dimensions are driven by physics/economics, not the roof proxy.

Architecture

Key models used in the pipeline:

ComponentModel
Self-consumptionHTW Berlin calibrated lookup table (not exponential)
Battery liftPower-law: 0.05 × battery^0.55 / R^0.35, capped at 85%
Economics20-year NPV with escalating electricity prices and 3% discount rate
Heat pump savingsGas cost displacement: old gas cost − new HP electricity cost (COP 3.0)
Realism scoringBrand prior + battery popularity + demand-proportional sizing + battery/module ratio
Balanced selection0.30 × NPV + 0.45 × realism + 0.25 × self-sufficiency
AI explanationLocal Llama 3.2 (3B) via Ollama, fallback to Claude API or template
